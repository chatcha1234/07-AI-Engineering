# Auto Update README Skill

## 🎯 จุดประสงค์ (Purpose)

เพื่อให้ไฟล์ `README.md` สะท้อนโครงสร้างโปรเจกต์ แพ็กเกจที่ใช้ และคำสั่งที่จำเป็นต้องรันอยู่เสมอ โดยที่ผู้ใช้ไม่ต้องคอยบอกให้อัปเดตเองทุกครั้ง

## ⚡ เงื่อนไขการทำงาน (Triggers)

ทักษะนี้จะทำงานอัตโนมัติ (Proactive) เมื่อเกิดเหตุการณ์ต่อไปนี้:

1. มีการเพิ่ม หรือลบแพ็กเกจ (เช่น การรันคำสั่ง `uv add`, `uv remove`, `pip install`)
2. มีการแก้ไขไฟล์จัดการ dependency เช่น `requirements.txt` หรือ `pyproject.toml`
3. มีการปรับแต่ง environment เช่น การเปลี่ยนเวอร์ชัน Python (`pyenv local`)
4. เมื่อมีการเพิ่มโค้ดหลัก (Entry point) ใหม่ๆ หรือเพิ่มโฟลเดอร์โครงสร้างที่สำคัญ

## 🛠️ ขั้นตอนปฏิบัติ (Actions)

1. **อ่านไฟล์ปัจจุบัน**: ใช้เครื่องมือ `view_file` อ่านโครงสร้างของ `README.md`
2. **วิเคราะห์การเปลี่ยนแปลง**: ตรวจสอบว่าคำสั่งใหม่ หรือแพ็กเกจที่เพิ่มมานั้น ควรจะอยู่ในหมวดหมู่ไหน (เช่น Dependencies, Project Setup หรือ Deployment)
3. **ทำการแก้ไข**:
   - ใช้เครื่องมือ `replace_file_content` หรือ `multi_replace_file_content` เพื่อแทรกข้อมูลเข้าไป
   - ใช้รูปแบบ Markdown ที่ถูกต้องเสมอ เช่น ใส่บล็อกโค้ด ```bash สำหรับ bash script
4. **แจ้งผู้ใช้**: รายงานสั้นๆ พร้อมกับผลลัพธ์ว่า "อัปเดต README.md ให้เรียบร้อยแล้ว"

## 📝 กฎการจัดรูปแบบ (Formatting Rules)

- ให้คงโครงสร้างเดิมของ `README.md` ไว้ให้มากที่สุด
- ถ้าคำสั่งไหนเป็นคำสั่งใหม่ ให้เขียนบรรยายสั้นๆ ด้วยว่าทำงานเพื่ออะไร
- ใช้ Markdown headings (##, ###) และ Bullet points เพื่อให้อ่านง่าย
