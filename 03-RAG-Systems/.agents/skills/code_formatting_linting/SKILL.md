# Code Formatting and Linting Skill

## 🎯 จุดประสงค์ (Purpose)

รักษามาตรฐานและคุณภาพของโค้ดในโปรเจกต์ Python ให้เป็นไปในทิศทางเดียวกัน ลดบั๊กที่อาจเกิดจากการพิมพ์ผิดหรือเว้นวรรคผิด

## ⚡ เงื่อนไขการทำงาน (Triggers)

1. ผู้ใช้ร้องขอให้ "จัดหน้าโค้ด" "ล้างโค้ด" หรือ "format code"
2. เมื่อผู้ใช้เพิ่งเขียนสคริปต์ Python หรือฟังก์ชันใหม่ที่ค่อนข้างยาวเสร็จสิ้น
3. สังเกตเห็นว่าโค้ดมีการเยื้อง (Indentation) หรือสไตล์ที่ผิดเพี้ยนไปจากปกติ

## 🛠️ ขั้นตอนปฏิบัติ (Actions)

1. แนะนำให้ผู้ใช้ติดตั้งเครื่องมือ formatting/linting หากยังไม่มี (เช่น `ruff` ซึ่งเร็วมาก และเข้ากับ `uv` ได้ดี) ด้วยคำสั่ง `uv add ruff`
2. เพื่อจัดรูปแบบโค้ด: แนะนำ/รันคำสั่ง `ruff check --fix .` และ `ruff format .`
3. แจ้งผู้ใช้ถึงผลลัพธ์ว่ามีไฟล์ใดที่ถูกปรับแก้ไปบ้าง
4. หากมีข้อผิดพลาด (Syntax error) ที่เครื่องมือแก้ไม่ได้ ให้ชี้แจงตำแหน่งและวิธีแก้ให้ผู้ใช้ทราบ
